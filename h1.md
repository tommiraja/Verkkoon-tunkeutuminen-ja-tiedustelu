# H1 Sniff

## Kotikone, Oracle VB Debian 13 Linux

## Tiivistelmät

Karvinen 2025, Wireshark basics: (https://terokarvinen.com/wireshark-getting-started/)

- Wireshark ohjelmaa käytetään verkon analysointiin, sen avulla voidaan siepata verkkoliikennettä ja tutkia sitä
- Verkkoliikenne sieppauksia voidaan tallentaa koneelle pcap-tiedostoina (packet capture file)
- Verkkoliikennettä tutkiessa Wiresharkilla on mahdollista nähdä yleiskuva sieppauksesta Statistics valikosta. Niistä nähdään mm. 
päätepisteet, hostit, I/O-kaaviot, jotka kuvaavat milloin verkkoliikenne oli aktiivinen ja kuinka kauan verkkopaketteja siepattiin, sekä protokolla hierarkia
- Liikennettä voidaan filtteröidä löytääkseen tietynlaista liikennettä

Karvinen 2025, Network Interface Names on Linux: (https://terokarvinen.com/network-interface-linux/)

- Verkkoliitännät ovat kuin verkkokortteja, mutta ei fyysisiä. Lyhenteet tarkoittavat seuraavaa: en = Wired Ethernet, wl ) WLAN, lo = loopback adapter
- Omat verkkoliitännät voidaan listata `ip a`, sekä `ip route` komennoin. Yleisiä liitäntä nimiä `wlp4s0 = Wifi card`, `enp1s0 = Wired ethernet card`
- Loopback adapteria käytetään yhteyksiin, joita ei reititetä tietokoneen ulkopuolelle, yhteys pysyy siis yksityisenä ja tietokone ikään, kuin kommunikoi itsensä kanssa. Yleinen loopback osoite on 127.0.0.1/8 ja IPv6 osoitteet ::1/128

### d) TCP/IP

- Tutkitaan yhtä siepattua pakettia ja osoitetaan TCP/IP-mallin neljä kerrosta siitä paketista
- Aloitetaan sieppaus `start capturing` -> CLI:ssä `nslookup google.com`
- Nslookup listaa tietyn verkkopalvelun serverin osoitteen

<img width="1047" height="631" alt="image" src="https://github.com/user-attachments/assets/f01ece5c-6cec-4bb8-b5dd-c795ae685b48" />

- Tuplaklikataan yhtä pakettia ja analysoidaan

<img width="747" height="455" alt="image" src="https://github.com/user-attachments/assets/bd4d08a8-767b-4b28-929a-65b2d8736152" />

- Huomataan viisi riviä Frame, Ethernet II, IPv4, UDP, sekä DNS
- Keskitytään neljään jälkimmäiseen riviin
- TCP/IP-mallissa neljä kerrosta ovat Network Access, Internet, Transport, sekä Application layer (https://www.geeksforgeeks.org/computer-networks/tcp-ip-model/)
- Ethernet II -> Link Layer = Vastaa tiedonsiirrosta verkkolaitteiden kautta, varmistaen että tiedot saavuttavat kohteen ja seuraavan laitteen
- IPv4 -> Internet Layer = Network access layer = Vastaa pakettien lähettämisestä, pakkaamisesta ja reitityksestä, mahdollistaa pakettien liikkumisen verkkojen yli saavuttaen oikean kohdelaitteen
- UDP -> Transport layer = Kuljetuskerros, joka vastaa datavirran segmentoinnista, kokoamisesta, mahdollistaa sovellusten välisen kommunikoinnin, se toteuttaa myös virheenkorjausta, jotta tiedot saapuvat virheettöminä maaliin (TCP & UDP)
- DNS -> Application layer = Tarjoaa käyttöliittymän sovellusten ja verkkopalvelujen välille (HTTP, FTP, SMTP, DNS)

i) Analysoi omaa liikennettä

- Aloitin enp0s3 verkkoliikenteen kaappauksen ja avasin selainikkunan, sitten suuntasin kurssimme kotisivuille
- Valitsen ensimmäiset 6 pakettia

<img width="1056" height="367" alt="image" src="https://github.com/user-attachments/assets/55a393bf-a09b-45e9-8b6b-cb566ab33786" />

- Kaksi ensimmäistä pakettia tekee DNS Standard queryn, eli koneeni kysyy DNS-palvelimelta IPv4 ja IPv6 osoitteet (A tarkoittaa IPv4 ja AAAA IPv6-osoitetta)
- Tämän jälkeen DNS-palvelin vastaa kyselyyn "Standard query response" antaen nimelle kuuluvan ip-osoitteen
- 5. paketin kohdalla muodostuu TCP-yhteys, joka nähdään protokolla kohdasta

<img width="604" height="75" alt="image" src="https://github.com/user-attachments/assets/855333f5-4054-40ac-aa7b-19ec5a942b6b" />

- Lähdeportti 59612 -> kohdeportti 443, tässä virtuaalikoneeni aloittaa TCP-yhteyden, joka kohdistuu porttiin 443. `SYN` tarkoittaa, että TCP-yhteyden muodostumisen aloitusta

- 6. paketti näyttää samalta, mutta portit ovat päinvastoin, joka tarkoittaa sitä, että portti 443 on vastaanottanut TCP-yhteyden ja vastaa asiakaskoneelle takaisin
- `SYN` & `ACK` tarkoittavat tässä tilanteessa sitä, että TCP-yhteys hyväksytään porttien välillä ja asiakaskoneen pyyntö vastaanotetaan
